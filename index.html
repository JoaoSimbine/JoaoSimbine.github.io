Processo

Para cada problema existe uma solução diferente.


Para a criação dos websites eu sigo um processo, ou seja, eu sigo um padrão para garantir que a solução que vai sair, seja a solução ideal para o problema do cliente. Agora segue uma breve explicação sobre o processo que uso para criar os websites:

01 - Briefing

O briefing é um documento que servirá como um guia para a execução de um projeto. Ele contém uma série de informações como: dados sobre a empresa, o mercado em que ela atua, o público ao qual ela se direciona e os seus objectivos com o projeto É aqui onde você senta comigo e diz qual é o seu problema, oque você deseja fazer.


Esta é a parte mais importante, pois é onde vamos estabelecer os requisitos básicos, é onde vamos definir os nossos principais objectivos.

02 - Wireframe

O wireframe se trata de um protótipo da página de um site ou aplicativo, ou seja, antes da elaboração do layout, é criado uma espécie de rascunho.


Com ele, é possível enxergar como o produto final vai ficar e, se tiver algo errado, dá tempo de mudar no meio do caminho, ajustando o esboço quantas vezes for necessário.

03 - Design Final

Nesta etapa é onde eu vou trazer o design final, como a aplicação/site vai ficar quando for programado.

04 - Desenvolvimento

Nesta etapa é onde a programação acontece. Aqui você só precisa ter paciência que dentro de 07 dias o website já estará completo.

05 - Revisões e Hospedagem

Esta é a última etapa, onde vamos fazer uma revisão do que foi feito e por fim hospedamos o site. Hospedagem de site é um serviço online que permite a publicação de um site ou aplicação na internet. Quando você adquire uma hospedagem, basicamente você está alugando um espaço dentro de um servidor.
